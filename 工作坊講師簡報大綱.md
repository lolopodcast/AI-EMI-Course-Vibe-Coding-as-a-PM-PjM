# 工作坊講師簡報大綱（A1-A6、B1-B6 全套濃縮）

> 用途：把整套工作坊（[工作坊設計文件](工作坊設計_AI-native產品開發工作坊.md)）濃縮成「每個 Session 該放哪幾張投影片、每張投影片放什麼內容」的大綱，供講師實際做成投影片檔案使用。
> 說明：這是**投影片大綱**，不是逐字講稿——完整口語講稿與互動細節請見 [SessionA5](SessionA5示範腳本_Agile概念導入.md)／[SessionB2](SessionB2示範腳本_EMI案例.md)／[SessionB3](SessionB3示範腳本_多重代理人.md) 三份逐分鐘腳本。全套建議 **57 張投影片**（含活動指示頁），可依實際排版拆併。

## 基礎模組（6 小時，共 30 張）

### Session A1｜開場 + 六層架構總覽（40 分鐘，4 張）
1. **標題頁**：當老師變成 PM / PjM——用 AI-native 流程打造課程數位產品
2. **為什麼老師要懂 PM/PjM**：三個現場卡點（EMI 學生跟不上／老師沒時間／系所缺工具）
3. **六層架構總覽**：Needs → Goals → Spec → Graphiti/Schema → AI Coding → Production（流程圖）
4. **建築比喻**：開咖啡廳對照表（需求／目標／規格／關聯／尺寸／施工）
- 🎯 活動指示頁：寫下你的專案宣言（一句話）

### Session A2｜Needs → Goals（70 分鐘，5 張）
1. **Why vs What Outcome**：兩者的差異示意
2. **Customer Needs 範例**：EMI 案例三個 Needs
3. **Product Goals 範例**：Goal + 可衡量指標對照
4. **常見錯誤示範**：Goals 寫成功能清單 vs 寫成結果指標（反例對照）
5. 🎯 活動指示頁：寫 3 條 Needs → 2-3 個 Goals（附指標）

### Session A3｜Open Spec（70 分鐘，5 張）
1. **使用者故事句型**：身為〔角色〕，我想要〔行為〕，以便〔目的〕
2. **完成的定義（Definition of Done）**：為什麼一定要附
3. **EMI 案例 Open Spec 範例表**：S1-S5 完整表格
4. **檢查標準**：這條故事，AI 看了會不會追問？
5. 🎯 活動指示頁：每組寫 5 條使用者故事＋DoD

### Session A4｜Graphiti + Schema（70 分鐘，6 張）
1. **Graphiti 概念**：關係地圖／連連看圖
2. **友善替代說法**：族譜類比、成績單欄位類比
3. **EMI 案例關聯樹範例**：Course/Module/Student/AI Coach
4. **Schema 概念**：資料規格表
5. **EMI 案例 Schema 範例**：Course/Student/Conversation 欄位表
6. **對照檢查**：關聯圖每條線,Schema 都有欄位支撐嗎？
- 🎯 選讀講義卡：[SessionA4講義卡](SessionA4講義卡_Graphiti三層記憶.md)（尾聲發放）

### Session A5｜Agile / PjM 必要概念導入（80 分鐘，8 張）
1. **為什麼需要 Agile**：規格寫完≠專案會自己跑下去
2. **Backlog**：待辦清單概念＋EMI 案例 Backlog 範例
3. **看板（Kanban）**：三欄示意＋EMI 案例卡片分佈
4. **衝刺（Sprint）**：概念＋EMI 案例 2-3 個衝刺分配示範
5. **完成的定義深化**：「做完了」認知落差反例
6. **同步會（Standup）**：3 分鐘同步會示範對話
7. **回顧會預告 ＋ MVP**：先求有再求好
8. **Agile 概念對照表**：對應到六層架構的哪裡
- 🎯 活動指示頁：各組完成初版看板

### Session A6｜收尾：基礎規格包（30 分鐘，2 張）
1. **基礎規格包內容清單**：Needs/Goals/Spec/Graphiti/Schema/看板
2. **銜接預告**：明天／下半場實施模組會做什麼

## 實施模組（6 小時，共 27 張）

### Session B1｜Claude Code 入門（40 分鐘，4 張）
1. **什麼是 Claude Code**：能讀懂自然語言、讀寫檔案、執行程式的 AI 協作工具
2. **角色對照**：老師→PM/PjM、TA→Vibe Coder、Claude Code→Implementer
3. **示範預告**：等一下會現場做給你們看
4. **操作原則**：先請它「複述理解」再動手

### Session B2｜Vibe Coding 實作（90 分鐘，6 張）
1. **EMI 案例規格包總覽**：Needs/Goals/Spec/Graphiti/Schema 一頁式回顧
2. **組合完整指令示範**：實際貼給 Claude Code 的 prompt 逐字稿
3. **清楚規格的產出結果**：對照 Definition of Done 打勾表
4. **模糊規格對照演示**：同一個專案,模糊指令會漏掉什麼
5. 🎯 活動指示頁：各組貼自己的規格包，實際操作
6. **Vibe Coding Log 提醒**：記錄用了什麼 prompt、AI 誤解了什麼
- 📄 完整逐分鐘腳本：[SessionB2](SessionB2示範腳本_EMI案例.md)

### Session B3｜多重代理人協作（60 分鐘，6 張）
1. **三個任務**：課程頁面／AI 助教問答／每週報表
2. **判斷平行 vs 相依**：回頭看 Graphiti 關聯圖找答案
3. **發派指令示範**：一句話請 Claude Code 平行處理兩件事
4. **審查型代理人**：先列問題清單、不直接動手改
5. **相依任務排序示範**：報表要等問答資料格式確定才開始
6. 🎯 活動指示頁：各組判斷自己專案的平行/相依任務
- 📄 完整逐分鐘腳本：[SessionB3](SessionB3示範腳本_多重代理人.md)

### Session B4｜交接文件撰寫工作坊（40 分鐘，3 張）
1. **交接文件範本總覽**：10 節內容一覽
2. **自我檢查清單**：交接文件範本文末的 6 條檢查
3. 🎯 活動指示頁：整理今天產出進範本

### Session B5｜分組演練：角色互換交接（90 分鐘，4 張）
1. **演練規則**：交接文件互換，禁止口頭補充說明
2. **任務**：只憑對方文件，用 Claude Code 延伸對方原型
3. **看板追蹤提醒**：新工作項目加進對方的看板
4. **Debrief 討論題**：文件哪裡看不懂？AI 誤解了什麼？

### Session B6｜成果展示 + 回顧會 + SOP 帶走（40 分鐘，4 張）
1. **展示規則**：每組 5 分鐘，原型＋交接文件
2. **Sprint Retrospective**：Keep / Problem / Try 三欄
3. **SOP 帶走清單**：完整交接文件＋空白範本＋回顧紀錄
4. **結尾頁**：致謝＋下一步／後續追蹤方式

## 給講師的簡報製作建議

- **視覺風格**建議延用 [工作坊行銷簡報.html](工作坊行銷簡報.html) 的「藍圖（Blueprint）」視覺主題，維持整套教材識別一致：深藍圖紙background＋淺色網格線＋accent 用工程圖常見的琥珀/橘色
- **每個 Session 開頭**建議固定放一張「今天走到哪裡了」的六層架構定位圖，讓學員清楚知道目前在整體架構的哪一層
- **案例一致性**：全程使用駱世民教授（Prof. Shihmin Lo）的 EMI 課程「國際策略管理（SMMC）」案例，不要中途換案例，降低學員的認知負擔
- 若要做成正式 PPTX 檔案，可以直接把這份大綱交給我，我可以逐頁展開成實際簡報檔
