## 0. 心理建設：為什麼要多花時間寫這份文件

先劇透一件事：**Session B5 會把你這份文件交給完全沒參與過程的另一組人，只憑文件接手**。如果對方看不懂、卡住了，代表這份文件現在還不合格。與其到時候才發現，不如現在花 40-60 分鐘，把它寫到「換一個人也看得懂」的程度。

這不是多餘的行政作業——**這就是整套工作坊在教的核心能力：把腦中的東西講清楚，讓不在場的人也能接手**。

### 2-3. Customer Needs／Product Goals／4. Open Spec／5. Graphiti／6. Schema

### 6.1 Graphiti↔Schema 交叉檢查（示範重點）
```
| Graphiti 關聯線 | 支撐類型 | 說明 |
|---|---|---|
| Course → Module | 直接外鍵 | Module.course_id |
| Course → Student | 隱含於範圍 | 系統目前只服務單一課程（SMMC），所有學生天生屬於這門課，不需要 course_id 欄位；若未來擴充成多課程平台才需要補 |
| Student → Conversation | 直接外鍵 | Conversation.student_id |
| AI Coach → Conversation | 隱含於範圍 | 系統只有一個 AI Coach，不需要額外欄位指定是哪一個 |
| WeeklyReport ← Conversation | 統計彙總 | WeeklyReport 是把一段時間內的 Conversation 依主題分類統計而成，不是外鍵指向 |
```
完整判斷原則見 [SessionA4自學指南_GraphitiSchema.md](SessionA4自學指南_GraphitiSchema.md) 第 3 節。

### 7. Vibe Coding Log（示範重點）
```
| 所屬回合 | 日期 | 對應 Spec | 分工代理人 | Prompt 摘要 | 產出結果 | 是否符合預期 |
|---|---|---|---|---|---|---|
| 回合1（清楚規格） | 07/09 | S1-S5 | 主對話 | 貼上完整規格包，請先複述理解 | 產出含大綱/雙語問答/報表雛形的原型 | 符合，5條DoD都對得上 |
| 回合1（模糊對照，僅示範用） | 07/09 | — | 主對話 | 故意用一句模糊指令重跑 | 缺雙語、缺對話紀錄、缺信心不足告知 | 刻意對照，證明規格清晰度的重要性 |
| 回合2（多重代理人） | 07/10 | S1+S2 | 探索型+實作型（平行） | 請同時處理課程頁面(A)與雙語問答(B) | 兩者皆完成，B產出對話資料格式如Schema所述 | 符合 |
| 回合2（審查） | 07/10 | S1,S2,S5 | 審查型 | 請檢查A、B是否符合DoD | 列出2點待修，已請B修正 | 已修正 |

```

### 8. 目前完成度
```
✅ 已完成：S1（課程頁面）、S2（雙語問答，模擬回覆）、S5（信心不足告知邏輯）
🚧 進行中：S3（對話紀錄，資料結構已定，尚未接真實資料庫）
⛔ 尚未開始：S4（報表——邏輯已寫但只用模擬資料測過，尚未用真實對話資料驗證）
⚠️ 已知限制／Bug：

  - 報表統計邏輯只測過 3 筆模擬資料，資料量大時是否正確尚未驗證
```

## 3. 自我檢查：對照範本文末的清單，逐條誠實回答

打開範本文末的「交接文件自我檢查清單」，**針對你自己剛寫好的文件**，逐條回答：

- [ ] 如果接手者完全不能問我，他能靠這份文件獨立完成下一步嗎？——**具體想像**：陳 TA 如果明天請假，換一個完全沒參與過的人接手，他看得懂「下一步該做 S2 的 API 串接」嗎？
- [ ] Product Goals 是不是寫成了「結果指標」，而不是「功能清單」？
- [ ] Open Spec 裡每一條，AI 讀了會不會需要再追問？
- [ ] Graphiti 裡畫出的每一條關聯，Schema 裡都有對應欄位支撐嗎？
- [ ] Vibe Coding Log 有沒有記錄「AI 誤解了什麼」，讓下一次少走冤枉路？
- [ ] 已知限制有沒有誠實列出，而不是只寫完成的部分？

👉 如果有任何一條答不出來或猶豫，回頭把那一節補得更具體，不要留到 Session B5 才被同儕發現。

## 4. 找碴練習：對照「模糊版」與「具體版」，校準自己的標準

自學時最大的風險，是「自己覺得寫得夠清楚，其實不夠」。以下用同一個項目對照兩種寫法：

| 項目 | ❌ 模糊版（不合格） | ✅ 具體版（合格） |
|---|---|---|
| 目前完成度 | 「大致做完了，還有一些小問題」 | 「S1、S2、S5 已完成；S3 資料結構已定但未接真實資料庫；S4 只測過模擬資料」 |
| 下一步建議 | 「繼續調整就好」 | 「優先把 S2 換成真實 API；S4 需要用 20 筆真實資料重新測試」 |
| Vibe Coding Log | 「有跟 AI 對話，做出東西了」 | 「用清楚規格 vs 模糊指令各跑一次，模糊版缺雙語/紀錄/信心告知三項功能」 |

**判斷原則**：如果一句話拿掉具體名詞（S1、S2、20筆資料、API）還能通順地讀完，通常就是模糊版。具體版一定會出現「數字、代號、專有名詞」。
