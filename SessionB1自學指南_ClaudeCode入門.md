# Session B1 自學指南：Claude Code 入門

> 對應：[工作坊設計文件](工作坊設計_AI-native產品開發工作坊.md) Session B1（原設計 40 分鐘，自學建議抓 50-60 分鐘，含安裝時間）
> 使用對象：完全沒寫過程式的老師／學生，**第一次**要在自己的電腦上安裝並操作 Claude Code
> 前置作業：建議先完成基礎模組（尤其是 Session A6 產出的「基礎規格包」），沒有規格包也可以先跟著本節走一遍最小範例
> 本文件風格：直接對你（自學者）說話，不是講師台詞——照著步驟做，每一步都有「怎麼確認我做對了」

---

## 學習目標

完成本節後，你應該能夠：
- 用一句話解釋 Claude Code 是什麼、能做什麼、不能做什麼
- 在自己的電腦上成功安裝並啟動 Claude Code
- 用中文或英文自然語言，讓 Claude Code 讀懂你要它做的事
- 完成第一個「規格 → 原型」的最小示範，並知道怎麼驗證它有沒有成功

---

## 0. 心理建設：你不需要會寫程式

先講清楚一件事：**接下來所有步驟裡，你唯一需要打字的地方，就是用中文或英文描述你要什麼**。看到「終端機」「安裝」這些字眼不用緊張——這比安裝一般的電腦軟體多不了幾步，而且只需要做一次。

回顧一下角色分工（見 [工作坊設計文件](工作坊設計_AI-native產品開發工作坊.md) 第 1.2 節）：
- 你是 Product Manager／Project Manager——負責講清楚要什麼
- Claude Code 是 Implementer——負責把它做出來

本節的目標就是把「Implementer」請到你的電腦裡，準備好等你下指令。

---

## 1. 什麼是 Claude Code（概念）

- **Claude Code 是一個安裝在你電腦上的 AI 協作工具**，透過「終端機」（Windows 叫「命令提示字元」或 PowerShell，Mac 叫「Terminal」）操作
- 它可以：讀懂你的中文／英文指示、直接在你電腦的資料夾裡讀寫檔案、執行程式、把結果告訴你
- 它**不是**：像 claude.ai 網頁版那樣「只能聊天、看不到你的檔案」的工具——Claude Code 是真的能動手操作你電腦裡檔案的版本
- 一句話記住：**claude.ai 網頁版是「顧問」，Claude Code 是「顧問＋動手做的助理」**

---

## 2. 安裝前準備

你需要：
- 一台 Windows 或 Mac 電腦，有網路連線
- 一個 Anthropic 帳號（跟你平常用的 claude.ai 帳號是同一個），登入時會需要用得到
- 確認自己的方案是否支援 Claude Code（免費／付費方案的額度與功能可能不同，建議先到 claude.com 官網確認目前的方案內容，因為方案與定價會隨時間調整，這裡不寫死數字，以免資訊過時）

> 💡 如果你完全不確定自己的帳號方案夠不夠用，這是唯一建議「找人問一下」的地方——其餘步驟都可以自己來。

---

## 3. 安裝步驟

### Windows
1. 按開始鍵，搜尋「PowerShell」，點開它（不需要用「系統管理員」身分，一般開啟即可）
2. 在跳出來的黑底／藍底視窗裡，貼上以下指令，按 Enter：
   ```
   irm https://claude.ai/install.ps1 | iex
   ```
3. 等它跑完（會看到一些安裝訊息跑過），跑完後這個視窗先不要關

### Mac
1. 按 `Command + 空白鍵`，輸入「Terminal」，打開它
2. 貼上以下指令，按 Enter：
   ```
   curl -fsSL https://claude.ai/install.sh | bash
   ```
3. 等它跑完，這個視窗先不要關

> ⚠️ 如果上面的指令出現「找不到指令」或類似的錯誤訊息，代表你的電腦可能需要先安裝 Node.js（另一個開發工具的基礎環境）。這時候最快的自救方式，就是把完整的錯誤訊息複製起來，拿去問 claude.ai 網頁版：「我在安裝 Claude Code 時看到這個錯誤，該怎麼辦：〔貼上錯誤訊息〕」，網頁版可以直接教你下一步。

---

## 4. 第一次啟動與驗證

1. 在同一個終端機視窗裡，輸入：
   ```
   claude
   ```
   按 Enter
2. 第一次執行會引導你登入——它會跳出一個網頁登入畫面，用你的 Anthropic 帳號登入並授權即可
3. 登入成功後，畫面會出現 Claude Code 的歡迎畫面，並且有一個可以打字的輸入框

**✅ 自我檢查點**：如果你看到歡迎畫面、可以打字，代表安裝成功。如果卡住了，把畫面上出現的文字複製起來，去問 claude.ai 網頁版，跟它說你在安裝 Claude Code 時卡在這一步。

---

## 5. 跟 Claude Code 說話的基本原則

不需要學任何程式語法，只需要記住三件事：

1. **用你自己的話講清楚你要什麼**，越具體越好（這正是基礎模組整套在練習的能力）
2. **可以請它「先複述理解再動手」**——輸入指令後，可以先加一句「先跟我確認你的理解，再開始做」
3. **可以隨時追問**——如果看不懂它做了什麼，直接打字問它：「你剛剛做了什麼？」「這個檔案是做什麼用的？」

## 6. 第一個練習：用 EMI 案例做一個最小示範

沿用 [SessionB2示範腳本_EMI案例.md](SessionB2示範腳本_EMI案例.md) 駱世民教授的 EMI 課程案例，做一個**極簡單、只是為了驗證安裝成功**的小練習（正式的完整規格包示範留到 Session B2）。

**步驟**：
1. 在終端機裡，先切換到一個你想放練習檔案的資料夾（例如桌面新建一個叫 `claude-練習` 的資料夾），輸入：
   ```
   cd 桌面/claude-練習
   ```
   （如果這個資料夾還不存在，先在電腦上手動新增一個空資料夾，再切換進去）
2. 輸入 `claude` 啟動
3. 貼上以下指令：
   ```
   請在這個資料夾裡建立一個簡單的 index.html 網頁，標題寫「國際策略管理（SMMC）課程網站」，
   內容只要有一句歡迎詞就好，先跟我確認你的理解，再開始做。
   ```
4. Claude Code 會先回覆它的理解，等你確認（直接回覆「可以，請開始」）
5. 完成後，去你剛才那個資料夾裡，應該會看到一個 `index.html` 檔案

**✅ 自我檢查點（怎麼確認成功）**：
- [ ] 資料夾裡真的出現了一個新檔案 `index.html`
- [ ] 用瀏覽器打開這個檔案，看得到「國際策略管理（SMMC）課程網站」的標題
- [ ] Claude Code 有先跟你確認理解，才動手做（如果它直接做了沒有先問，也沒關係，代表你的指令已經夠清楚）

如果三項都打勾，恭喜，你已經完成了人生第一次 Vibe Coding。

## 8. 常見問題自助排除（沒有講師在旁時）

| 狀況 | 建議自救方式 |
|---|---|
| PowerShell／Terminal 打開後看不懂畫面 | 這是正常的，那個黑底／藍底畫面就是用來打字下指令的地方，不用怕 |
| 安裝指令跑出錯誤訊息 | 複製完整錯誤訊息，貼去問 claude.ai 網頁版：「我在安裝 Claude Code 時看到這個錯誤，該怎麼辦」 |
| `claude` 指令找不到 | 通常是安裝過程沒跑完，或需要關掉終端機重新打開一次再試 |
| 登入卡住、網頁打不開 | 確認網路連線正常，並確認用的是你有效的 Anthropic 帳號 |
| 練習做出來的網頁打不開／空白 | 確認資料夾路徑對不對、`index.html` 檔案是否真的存在；可以把情況描述給 claude.ai 網頁版，請它幫你判斷 |
| 完全卡住、上面都試過還是不行 | 這是唯一建議暫停、去找 TA 或同事協助的情況——不需要獨自卡超過 15-20 分鐘 |

👉 這份自救表的精神，跟整套工作坊反覆強調的一樣：**遇到卡點，先具體描述問題（給 AI 或給人），而不是憑空重試**。
