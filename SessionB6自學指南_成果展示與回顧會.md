## 學習目標

- 用 2 分鐘講清楚自己做出來的原型
- 做一次誠實的個人回顧（Keep／Problem／Try），檢討流程而非檢討自己

## 1. 自我展示：錄一段 2 分鐘的成果發表（15 分鐘）

現場工作坊是「每組 5 分鐘上台展示」，自學版改成**對著手機或電腦錄一段 2 分鐘的自我介紹**——不是為了給別人看，是因為**逼自己講清楚，會逼出你自己都沒發現的邏輯漏洞**。

錄之前，先照這個架構打草稿：
```
1. 我的專案宣言是：__________（Session A1）
2. 我最終做出來的原型是：__________
3. 對照 Definition of Done，哪幾條真正做到了：__________
4. 如果還有下一步，我會優先做：__________
```

錄完後**自己重看一次**，觀察：
- 有沒有哪一段講得結結巴巴、講不清楚？——那通常就是你自己也還沒完全想清楚的地方
- 這段話，如果講給一個完全不懂這個專案的人聽，他聽得懂嗎？

**✅ 自我檢查點**：如果重看時發現某一段講不清楚，回頭補強對應的文件章節，而不是只怪自己「口條不好」。

> 💡 如果想更進一步，也可以用工作坊行銷簡報那套螢幕錄影方式（Xbox Game Bar／`Win+G`），實際錄下你操作原型的畫面，這樣以後回顧或跟別人分享都更直觀。

## 2. 一人版回顧會：Keep / Problem / Try（20 分鐘）

現場版本的回顧會本來就是「自己誠實反思」為主，一人自學做起來其實差異不大——**重點是要有具體的問題引導，不要只寫「還不錯」這種空泛的話**。

針對整個 12 小時（或你目前為止走過的部分），誠實回答：

### Keep（哪裡做得好，下次要繼續）
```
- 哪一個 Session，你覺得自己做得特別扎實？為什麼？
- __________
```

### Problem（哪裡卡住了，值得檢討）
```
- 哪一個 Session，你花的時間比預期久？卡在哪裡？
- 如果 Session B5 的測試裡，AI 對某個地方一直追問，那個地方是什麼？
- __________
```

### Try（下次要試試看什麼）
```
- 如果重來一次，你會在哪個環節換一種做法？
- 下一個專案，你會提早準備什麼（例如提早想清楚 Definition of Done、提早畫 Graphiti）？
- __________
```

**⚠️ 重要提醒**：回顧會的精神是**檢討流程，不是檢討自己「不夠聰明」或「學得慢」**。如果寫出來的東西偏向自我批評，換個角度重寫成「流程哪裡可以調整」。

## 3. SOP 帶走清單（10 分鐘）

整理以下三樣東西，存放在你未來會回頭找得到的地方（例如同一個資料夾）：

- [ ] 完整交接文件（已經過 Session B5 測試修正的版本）
- [ ] 一份空白交接文件範本（[交接文件範本_AI-native專案.md](交接文件範本_AI-native專案.md)），下一個專案直接複製使用
- [ ] 這次的 Keep／Problem／Try 回顧紀錄——**下一次開新專案前，先重讀一次這份紀錄**，比重新學一次整套工作坊更有效率

---

## 4. 自我檢查清單（完成整套工作坊前，全部打勾）

- [ ] 我錄了一段 2 分鐘展示，並重看過一次
- [ ] 我完成了 Keep／Problem／Try，且內容具體、不是空泛的感想
- [ ] 我把三樣 SOP 帶走清單都整理好，放在容易找到的地方
- [ ] 我知道下一次要做新專案時，該從哪一份文件、哪一個 Session 重新開始
