# Session A1 自學指南：開場 + 六層架構總覽

> 對應：[工作坊設計文件](工作坊設計_AI-native產品開發工作坊.md) Session A1（原設計 40 分鐘，自學建議抓 40-50 分鐘）
> 前置作業：無，這是整套工作坊的第一節
> 參照文件：[AI-native開發流程_KnowledgeBase.md](AI-native開發流程_KnowledgeBase.md) 第 1-2 節
> 本文件風格：直接對你（自學者）說話——照著步驟做，每一步都有「怎麼確認我做對了」

---

## 學習目標

完成本節後，你應該能夠：
- 說出六層架構的六個名稱與各自回答的問題
- 解釋為什麼這個順序很重要（不能跳著做）
- 說出老師在這套架構裡的角色轉變
- 寫出自己專案的「一句話專案宣言」

---

## 0. 暖身：你不需要會寫程式，也能主導開發

在動手之前，先確認一件事：**接下來十二小時要學的東西，完全不需要寫程式**。你會學到的是「怎麼把一個模糊的教學想法，講清楚到 AI 聽得懂」，動手把程式碼寫出來的是 AI。

## 1. 六層架構總覽（15 分鐘）

打開 [AI-native開發流程_KnowledgeBase.md](AI-native開發流程_KnowledgeBase.md) 第 1.2 節，看這張表：

| 層級 | 英文 | 核心問題 |
|---|---|---|
| 1 | Customer Needs | 為什麼要做？ |
| 2 | Product Goals | 要達成什麼結果？ |
| 3 | Open Spec | 系統必須做什麼？ |
| 4 | Architecture／Graphiti | 彼此如何連結？ |
| 5 | Schema | 資料長什麼樣？ |
| 6 | AI Coding → Production | 誰把它寫出來？ |

**自我測驗**：蓋上這張表，試著憑記憶把六個層級的名稱和對應的問題填出來。填不出來的地方，回頭再看一次那一層。

### 建築比喻（幫助記憶）

把整套架構想成蓋一間咖啡廳：
```
客戶需求 → 我要開咖啡廳
產品目標 → 50 個座位、可外帶
Open Spec → 要有吧檯、廚房、洗手間
Graphiti  → 吧檯和廚房怎麼連接、排水管怎麼走
Schema    → 每個房間尺寸、桌椅數量、插座位置
最後才施工
```

**✅ 自我檢查點**：你能不能用自己的話，把「蓋咖啡廳」換成另一個你熟悉的例子（例如辦一場活動、開一堂課），重新講一次這六層？如果講得出來，代表你真的理解了，不是死背。

## 2. 為什麼順序很重要（10 分鐘）

六層架構不是隨便排的，是有邏輯順序：
- 前三層（Needs → Goals → Spec）是**人類定義問題**的區塊
- 第四、五層（Graphiti、Schema）是**人類定義結構**的區塊，而且兩者會互相疊代、來回修正
- 最後一層（AI Coding）才是**AI 執行**的區塊

👉 **常見誤解**：以為六層是嚴格線性流程，走完一層才能走下一層。實際上只有前三層與最後兩層較線性；Graphiti 與 Schema 之間需要來回修正——這在 Session A4 會親自體驗到。

**自我測驗**：如果一開始沒想清楚 Customer Needs 就直接叫 AI 寫程式，會發生什麼問題？（提示：想想看，AI 要怎麼知道「做到什麼程度才算做完」）

## 3. 角色轉換（10 分鐘）

六層架構帶來的最大轉變，是角色：

| 傳統角色 | 這套架構裡的角色 |
|---|---|
| 老師 | 兼任 Product Manager（定義做什麼）與 Project Manager（安排何時由誰做） |
| TA／協作教師 | Vibe Coder（實際跟 Claude Code 對話產出程式碼） |
| Claude Code | Implementer（把規格變成可運作的產品） |

**自我測驗**：如果你以前覺得「我不會寫程式，所以做不了數位產品」，現在請用一句話重新描述你的角色。

## 4. 動手練習：寫下你的專案宣言

本套工作坊全程會用駱世民教授的 EMI 課程「國際策略管理（SMMC）」雙語 AI 助教作為範例案例，它的專案宣言是：

> 「我想幫我的 EMI 課程『國際策略管理（SMMC）』做一個雙語 AI 助教，讓學生下課後也能得到解答。」

**現在輪到你**：用一句話寫下你自己想做的課程數位產品。格式參考：「我想幫我的〔課程名稱〕做一個〔什麼東西〕，讓〔誰〕能〔達成什麼〕。」

寫下來：
```
我的專案宣言：
_______________________________________________
```

**✅ 自我檢查點**：
- [ ] 這句話有沒有具體到「別人看了就知道你想做什麼」，而不是「我想用 AI 做點什麼」這種空泛的說法
- [ ] 這句話裡有沒有出現「誰」（受益對象）

---

## 5. 找碴練習：校準「夠具體」的標準

| ❌ 太空泛 | ✅ 夠具體 |
|---|---|
| 「我想用 AI 幫我的課程加點什麼」 | 「我想幫我的 EMI 課程做一個雙語 AI 助教，讓學生下課後也能得到解答」 |
| 「我想做一個網站」 | 「我想幫〔課程〕做一個線上作業繳交系統，讓助教不用再手動收信件附件」 |

**判斷原則**：如果拿掉「誰、什麼、為什麼」任何一項，句子還是通順地讀完，通常就是太空泛。
